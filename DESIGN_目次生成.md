# DESIGN_目次生成.md

縦書き小説の目次（章立て）生成を「かなり正確」にするための設計。
実測サンプル: `黒牢城_米澤穂信_vision.pdf`（歴史小説525P・柱型）、
`ソフロニア嬢、倫敦で恋に陥落する_ゲイル・キャリガー_START.pdf`（章扉型・旧OCR417P）。
GOAL ePub の nav が正解。

---

## 0. 結論（一次ソースの反転）

現行 `refine_headings_with_toc` は**印刷目次を「既に良い本文見出しを磨く辞書」**
としてしか使わない。だが実測では**印刷目次ページのOCRは本文の章頭見出しより
一貫して正確**で、本文検出が崩れると良い目次まで道連れに棄却される（下記 §1）。

→ **印刷目次を章立ての一次ソースに格上げする**。目次を構造ベースで自力検出し、
章題の語彙を目次から確定し、本文側は「各章がどこから始まるか（位置）」だけを
供給する。位置信号は本の型で2種類ある（§2）。

---

## 1. 現状の証拠（両書とも「良い目次を下流が壊す」）

### 1.1 黒牢城 — 完璧な目次を `is_junk_line` が破壊

PDF p.4 の目次は縦行として**完全に**取れている（`--inspect 4` 実測）:

```
目次 / 序章因 / 第一章雪夜灯籠 / 第二章花影手柄 / 第三章遠雷念仏 /
第四章落日孤影 / 終章果 / 解説 / マライ・メントライン
```

しかし最終テキストには `序章因 / 終章果 / 解説 / マライ` しか残らない。
**第一〜四章が消える**。原因は `is_junk_line`（jisui2epub.py:510-517）:
「かな0・6字以上」を挿絵ノイズとして落とす規則が、歴史小説の四字熟語章題
（雪夜灯籠=7字かな0）を誤爆する。`序章因/終章果`(3字)・`解説`(2字)は短くて生存。
`assemble_text` の縦行ジャンクフィルタ（:1191）で落ちるため、`_toc_entry_of_line`
に届く前に消滅している。実測:
`is_junk_line("第一章雪夜灯籠")=True`／`_toc_entry_of_line("　第一章雪夜灯籠")=None`。

さらに黒牢城の**柱（章題を横書きでページ上部に印刷）がノンブルと融合**して
（p.55 柱=`57第一章夜`＋ルビ`灯`、p.57 柱ルビ=`第一`/`雪夜灯`）、`classify_marginals`
経由でページ毎に別々の偽見出しに化け、本文段落を分断する。実測（現行出力）:

```
…本来灯火を置く火袋には、ま      ← 段落が
［＃「57第一章夜」は中見出し］      ← 柱＋ノンブルの偽見出しで
だ何も置かれていない。            ← 分断される（「ま|だ」）
```

現行の章1見出し検出は `33 一章雪 / 57第一章夜 / 69 第一章雪 / 73章 雪 /
107章 雪夜灯籠 / 117 夜籠 / 129第一章雪夜 / 133章 雪灯籠` と**同一章の柱が
8個以上の別見出し**として乱立。巻末は `参考文献覧 / 角川文庫発刊に際して /
米澤穂信 / 帯広告blob` が見出しに昇格。GOAL は7項目
（序章因・第一〜四章・終章果・解説）。

### 1.2 ソフロニア — きれいな目次を「照合ゲート」が棄却

印刷目次（p.8前後）は良好に抽出されている（現行 sofro_now.txt 実測）:

```
危機その二舞踏会の侵入者刃 / 危機その三バラのなかの敵 / … /
危機その十六｜空強盗《フライウエイマン》を逃れ、火のなかへ卸 /
（もはや）危機（ではない）その十七ついに全員が… / それから誠 / とがき
```

各行末の1字ジャンク（刃/の/蝿/嘘…）はページ数列のOCR化け。危機その一だけ欠落。
これを `refine_headings_with_toc` が**使わない**。目次ページ採用条件が
「既検出の本文見出しと difflib0.75 で**3件以上一致**」（:1543-1547）。だが本文見出しは
章扉の飾り帯で `6爾弓言､6　危機その一人格交換一` のように破損しており、一致は
2件程度で3件に届かず、**良質な目次が丸ごと棄却**される（鶏と卵）。GOAL は20項目
（危機その一〜十七・それから・〈英国空中学園譚〉小事典・訳者あとがき）。

## 2. 本の2類型（位置信号の違い）

- **章扉型（ソフロニア・既存8冊の多く）**: 各章の先頭に大活字の章題（章扉）が
  本文中にある。`is_big`／横書き見出しで検出済み（破損しつつも位置は出る）。
  → 位置は本文見出しから、語彙（表記）は目次から。
- **柱型（黒牢城）**: 章扉が無く、章の識別は**柱（running header）だけ**が
  ページ毎に担う。柱はノンブル融合・毎ページ破損。序章は柱も無く本文が直に始まる。
  → 位置は**柱の変化**から、語彙は目次から。章扉型の位置検出は空振りする。

**統一データ構造**: 各本文ページに1つの「**章ラベル**（このページが属する章）」を
与え、印刷目次の章リストへ最近傍分類する。章の開始＝ラベルが変わる最初のページ。
ラベル源は型により章扉見出し or 柱。目次が語彙を固定するので、破損した柱・見出しも
「どの目次エントリに一番近いか」の分類問題に落ちる（クリーニング不要）。

## 3. アルゴリズム

### A. 純漢字章題の保護（`is_junk_line`／`_toc_entry_of_line`）

`is_junk_line` の「かな0・記号ありor6字以上→ジャンク」規則に**免除を追加**:
先頭が章マーカー（`_CHAPTER_HEAD_RE`＝`第N章|序章|終章|プロローグ|エピローグ`）
の行はジャンクにしない。実測で `第一章雪夜灯籠` 等が救われ、挿絵ノイズが
`第…章` で始まる確率は無視できる（章/話/部マーカー必須でガード）。
縦書き本文中に純漢字4字熟語の章題が出ても救えるので副作用は良性。
`_toc_entry_of_line` は is_junk_line 経由なので同時に直る。

**リスク**: 全域で効くため、稀に「第○章」風の挿絵ジャンクを拾う可能性。
→ 免除は「`_CHAPTER_HEAD_RE.match` かつ `bad文字率≤0.2`」に限定（記号だらけの
偽物は従来通り落ちる）。

### B. 目次ページの構造ベース検出（一致数ゲートの撤廃）

「既検出見出しと3件一致」を捨て、**セグメント内部の規則性**で判定する。
既知の誤検出事故（書を捨てよの詩集ページ・霧の本文半分消失）を防ぐため、
前付け限定などの位置ガードは**維持**した上で、採用条件を次に置換:

セグメント（改ページ・見出しで区切った範囲）が目次であるのは、**前付け内**
（先頭15% かつ 4番目の見出しより前）で、有効エントリ（`_toc_entry_of_line`）が
3件以上あり、かつ次のいずれか:

1. **番号の単調増加列を含む**: エントリの先頭章番号（`_LEADNUM_RE`＋漢数字
   `第一/序/危機その一` を数値化）が **3個以上、連続する増加列**をなす
   （黒牢城=1,2,3,4／ソフロニア=2,3,…,17）。詩行・広告・本文はこの列を作らない
   ＝最強の弁別子。
2. **目次ラベル（`_TOC_LABEL_RE`）が同一ページにある**（黒牢城は `目次` あり）。

上限はセグメント3のまま。1と2の両方があれば確度は極めて高い。番号列を作れない
本（`まえがき/序/本編` 等かな見出しのみ）は従来の difflib 経路にフォールバック
（後方互換）。

**漢数字番号のパーサ拡張**: `_LEADNUM_RE` は算用/全角数字のみ。黒牢城
`第一章`・序/終、ソフロニア `危機その十七` を数値化する漢数字パーサを新設
（`一二三四五六七八九十` の合成、`序=0` 相当・`終=∞` 相当は番号列から除外して
アンカーにしない）。

### C. 目次駆動の章頭位置探索（完全被覆）

目次エントリを**読み順のまま**章リストとし、各エントリの本文開始位置を決める:

1. まず検出済み本文見出しへ貪欲マッチ（現行 §2 の greedy を流用）。
2. マッチしないエントリは、**改ページ直後の本文先頭行**を difflib0.85 で探す
   （現行 欠落章回収 を全エントリに拡張。上限を5→目次エントリ数に緩和）。
   目次は信頼するが、位置探索は誤ると本文構造を壊すので改ページ隣接＋高しきい値は
   維持。
3. **章頭位置は単調増加を強制**（エントリiの位置 > エントリi-1の位置）。同題が
   複数箇所に出る本での誤アンカーを防ぐ。
4. 章扉が無い先頭章（黒牢城 序章）は、前付け（目次ページ）直後の最初の本文
   改ページへアンカー。柱型では D の柱ラベルが優先する。

### D. 柱ラベルの追跡と偽見出しの畳み込み（柱型の主機構）

柱がノンブル融合で毎ページ偽見出し化する問題を、**目次を語彙とする分類**で解く:

1. 各ページの柱候補テキスト（`classify_marginals` が柱/見出しに回した横行・
   ページ端縦行）から先頭ノンブル（`\d+` プレフィクス）を剥がす。
2. 目次章リストへ最近傍分類（`norm_hashira` 正規化 + difflib、しきい値0.5程度、
   同点は前ページのラベルを継続＝ヒステリシス）。閾値未満は「柱なし」。
3. **ページ列に対し章ラベルを前方充填**（柱の無いページは直前ラベルを継承）。
4. **章開始＝ラベルが変わる最初のページ**。ここでのみ目次のクリーン章題で
   `emit_heading` し、それ以外の柱occurrenceは見出しにしない（＝段落分断を停止）。

これで黒牢城の `57第一章夜 / 107章 雪夜灯籠 / …` は全て「第一章 雪夜灯籠」章の
本文ページと分類され、見出しは章1の最初の1回だけになる。柱のクリーニングは不要
（目次が語彙を持つ）。

**章扉型との統合**: 章扉見出しがあるページはラベル源として柱より優先。柱型/
章扉型の判定は「柱候補が全ページの一定割合以上に出て、かつ複数の目次エントリに
分散分類されるか」で自動判別（柱＝書名で一定＝1エントリに集中する本は柱型でない）。

### E. 巻末見出しの制限

最後の番号付き章（アンカー最大番号）より後ろの見出し候補は、次のいずれかで
なければ**採用しない**:

- 印刷目次に載っているエントリにマッチする、または
- `_UNNUMBERED_RE`（解説/あとがき/訳者/エピローグ/付録/年表/索引 …）に当たる。

これで `角川文庫発刊に際して`・`米澤穂信`（著者名）・帯広告blob は落ちる
（`_UNNUMBERED_RE` 非該当）。
`参考文献（一覧）` は `_UNNUMBERED_RE` に当たるが**印刷目次には無い**ため、
「目次にある or 後付け語」だけでは残ってしまう。GOAL は 参考文献 を nav に
含めない。

**【決定 2026-07-22】TOC一次ソースの原則を優先し既定降格**する。巻末で採用する
のは「印刷目次に載る章」＋「目次末尾以降に出現する **解説・あとがき・訳者・
おわりに・エピローグ**」のみ。参考文献/初出/謝辞/付録/年表/索引は目次に無ければ
既定で降格（GOAL に一致させる）。フラグでの残置オプションは今回設けない
（必要になれば後日 `--keep-backmatter` を追加）。実装では巻末許可語を
`_UNNUMBERED_RE` そのものではなく、上記の**後付け「章」語だけの絞り込み集合**
（`_BACKMATTER_KEEP_RE`）として新設する。

## 4. データフロー（触る関数）

```
assemble_text
  ├ A: is_junk_line 免除（章マーカー始まり）           … 縦行フィルタ:1191, 目次パーサ
  └ 従来どおり見出し候補（章扉/柱由来）を out に埋める
refine_headings_with_toc   ← ここを目次一次ソース化して再構成
  ├ B: 構造ベースの目次セグメント検出（番号単調増加＋ラベル）
  ├ 目次エントリ = 章リスト（読み順）
  ├ C: 各エントリの本文位置決定（既検出見出し→改ページ隣接探索、単調増加）
  ├ D: 柱型なら 柱ラベル追跡で章境界を決め、目次語彙で emit
  ├ E: 巻末制限（目次外＆非後付け語を降格）
  └ _renumber_headings（既存の連番内挿はそのまま活用）
```

`classify_marginals` は基本据え置き（柱候補の受け渡しのみ D で利用）。柱型判定と
柱畳み込みは `refine_headings_with_toc` 側に閉じ込め、他経路に副作用を出さない。

## 5. 誤検出ガードとの両立（既知事故）

- **書を捨てよ（詩集ページを目次誤認）／霧（本文半分を目次誤認）**: 番号単調
  増加列＋前付け限定で守る。詩行・本文は連続章番号列を作らない。B のフォール
  バック（difflib経路）も前付け限定のまま。
- **書名と同名の表題章（書を捨てよ）**: 現行 §6 の「目次に無い場合のみ降格」を維持。
- **グリック 第1部/第2部の2系列連番**: 番号単調増加の判定は「非減少かつ+1刻みの
  部分列が3以上」で見るので、部が変わってリセットする列も既存の
  `_renumber_headings` のアンカー方式（複数アンカー）で吸収。B の目次検出は
  「増加部分列が1本でも3以上あれば可」とし2系列を弾かない。

## 8. 実装結果（2026-07-22）

Stage 1〜3 を実装。回帰は temp_sample 9冊（グリック/ほんもの/伯爵夫人/書を捨てよ
SUPERFINE/タイム・リープ上/霧scansnap/30秒DS横書き/地下室vision/蘇我氏vision）。

- **Stage 1（A: 純漢字章題保護, B: 構造ベース目次検出）**: `is_junk_line` に
  `_CHAPTER_HEAD_RE` 始まりの免除を追加。目次採用を「章序数の+1連番列（`_max_
  increasing_run`）または目次ラベル＋エントリ3以上」に置換し、破損本文見出しとの
  一致数ゲートを撤廃。→ 黒牢城の目次から第一〜四章が復活、黒牢城・ソフロニアとも
  目次ページを検出。
- **Stage 2（C: 目次駆動の位置補完, E: 巻末制限）**: 回収上限を撤廃し読み順
  単調性を強制。巻末は正のジャンク署名（`_BACKMATTER_DROP_RE`＝参考文献/発刊/
  定価/著者名/長文帯等）でのみ降格し、後付け章語（`_BACKMATTER_KEEP_RE`）と
  壊れた実章（終章「果」）を保護。著者名照合はNFKC正規化。→ 黒牢城 巻末4件・
  グリック1件・ソフロニア2件の巻末ジャンクを降格。
- **Stage 3（D: 章扉検出＋柱の畳み込み＋序章/解説回収）**: 実装で判明した
  最重要事実＝**黒牢城は各章の章扉ページ（章題そのものが改ページに挟まれた
  独立本文行「第二章花影手柄」）を持ち、これが柱より正確な章境界**。よって:
  - **章扉検出（最優先）**: 改ページ隣接＋目次エントリへ0.88以上一致＋読み順
    単調の独立本文行を章扉とみなし、その位置へ正確に章見出しを立てる（`tobira`）。
    既検出見出しは head_set 除外で拾わない（通常本の重複昇格防止）。
  - **柱の畳み込み**: 章扉を持つ章の柱見出しは全てノイズとして畳む。章扉が無い
    章（終章・単発）は柱の最初の出現を代表にし、段落途中なら直近の文末境界へ
    挿し直す（`_place_rep_heading`・`_collapse_heading_block`）。
  - **序章/解説回収**: 序章型の先頭章は本文冒頭へ前置、後付け章（解説）は
    接頭辞一致で本文を残して前置。
  - **柱畳み込みの発火ゲート（誤爆防止が最重要）**: 「同一目次章が**先頭ノンブルの
    異なる3つ以上**の見出しに claim され、かつ**少なくとも1つがOCR破損**
    （entry一致<0.85）」の柱融合固有シグネチャのみ。候補も先頭ノンブル付きに限定。
    これで**正当な多部構成章**（グリックの「ドブネズミのガンバ(1)(2)」「戦い
    (1)(2)(3)」＝クリーンな連番章）や**章内反復セクション**（30秒DSの用語解説）を
    誤って畳まない。→ 実測: 発火ゲート緩いと guri 58→31・ds30 43→17 と実章大量
    消失。ゲート厳格化で両書とも非発火（改善のみ）に。
  - → 黒牢城が **序章因/第一章雪夜灯籠/第二章花影手柄/第三章遠雷念仏/
    第四章落日孤影/終章果/解説 の7章＝GOALと同一の章立て**、各章扉位置に正確配置。
- **【試行して撤回】章題修復の共通接頭辞（LCP）化**: 見出しと目次エントリの
  共通接頭辞をクリーン章題として採用する案を試したが、正しい語尾を誤った目次に
  合わせて切り詰める退行を確認して撤回（honmono「魔法都市マジェイア」→
  「マジェイ」＝目次側が誤る マジェイァ に引きずられる／グリック「ドブネズミの
  ガンバ(1)(2)」の部番号消失）。1字OCRゴミの切詰めと有意な語尾の切詰めが
  長さで区別できないため。ソフロニアの重破損章扉の清書は章扉OCR改善（Phase 2）に
  委ねる。
- **ePub生成部（`parse_aozora_text`）**: 章扉の前にある見出しなし前文
  （挿絵・章内改ページの続き）を独立エピソード化して目次に「第N話」を量産して
  いたのを、見出しなしセクションと同様に前エピソードへ連結する方式に統一。
  → 黒牢城 nav の「第5話」等の見出しなしノイズ項目が消滅。

### 既知の限界（残課題）

- **章扉の無い柱型本**: 章扉ページを持たず柱だけが章題を運ぶ本では、章頭が
  柱の初出（各章の最初の奇数ページ＝真の章頭より最大1〜2段落後ろ）になる。
  完全な配置にはページ構造を assemble_text 段階で使う上流実装が必要。
- **ソフロニアの章扉検出**: 危機その一〜十七のうち約半数は章扉（飾り帯付き）の
  OCRが崩れて見出し検出も章扉検出も漏れる（章題テキストが読めない）。
  Stage 1〜3 で巻末ジャンク除去・目次採用・表記修復・照合は改善したが、
  崩れた章扉ページの回収は章扉OCR自体の改善（再OCR/Phase 2）が必要。

# 自炊変換GUI（Windows）設計書

作成: 2026-07-20
対象ツール: jisui2epub.exe / vision_reocr.exe / docai_reocr.exe（本リポジトリ）
＋ manga_p2epub.exe（../mangaP2ePub リポジトリ）

## 1. 目的

自炊PDF→ePub変換の3ツールを、コマンドラインを使わずWindows上のGUIで
実行できるようにする。対象ユーザーは開発者本人（将来的には配布も視野）。
弱視・老眼のユーザーが使うため、**大きな文字・少ない操作手順・迷わない画面**
を最優先の設計原則とする。

現状の運用と課題:

- 3ツールとも PyInstaller製の単体exe（`forwindows/`）が存在し、PowerShellから
  オプションを付けて実行している
- 課題: (1)オプションの記憶・タイプが面倒（--horizontal --ruby drop 等の
  組み合わせ）、(2)vision_reocr のGCP認証環境変数の設定が煩雑、
  (3)長時間ジョブ（再OCR）の進捗・中断・再開の管理、(4)校正ループ
  （変換→txt修正→--from-text 再生成）の往復

## 2. 方式の比較と選定

| 方式 | 実装量 | 依存 | 評価 |
|---|---|---|---|
| **A. Tkinter製ランチャーGUI（採用）** | 中（1ファイル） | 標準ライブラリのみ（D&Dのみ小依存） | exe化容易・既存3exeを子プロセス起動する疎結合。CLIは無改造 |
| B. ツール本体へのGUI組み込み | 大×3 | — | 3ツールそれぞれに手が入り保守が乖離。不採用 |
| C. ローカルWebアプリ（ブラウザUI） | 大 | Flask等＋ブラウザ起動 | 単一ユーザーのデスクトップ用途には過剰。不採用 |
| D. バッチファイルへのドラッグ&ドロップ | 極小 | なし | オプション選択・進捗表示・GCP設定ができない。**補助手段として併用**（§8） |
| E. PySide/wxPython製GUI | 中 | 重い（+40MB級） | 見た目は良いがexe肥大とライセンス管理が増える。不採用 |

**採用: A**。理由は「CLI群が正であり続ける」こと。GUIは公開済みオプションを
組み立てて子プロセスを起動するだけの薄いランチャーとし、ツール本体の検証
実績（回帰テスト・ヒューリスティック）に一切影響を与えない。ツールが
クラッシュしてもGUIは生き残り、ログを表示できる。Tkinterは Python 標準添付で
PyInstaller との相性も良く、配布は既存と同じ「forwindows/ にexeを1本足す」
だけで済む。

## 3. 全体構成

```
jisui_gui.py（本リポジトリに新設・単一ファイル）
  ├─ 画面: Tkinter + ttk（DPIスケール対応、フォントサイズ設定つき）
  ├─ 実行: subprocess.Popen で既存exe/pyを起動、stdout/stderrを
  │        ワーカースレッドで読み取り → queue → after()ポーリングでログ欄へ
  ├─ 設定: %APPDATA%\jisui2epubGUI\settings.json
  └─ 配布: PyInstaller --onefile --noconsole → forwindows/jisui_gui.exe
```

ツールの解決順序（設定で個別上書き可能）:

1. GUI自身（exe）と同じフォルダの `jisui2epub.exe` 等
2. settings.json に保存されたパス
3. 開発時（.py実行時）は `python jisui2epub.py` 等にフォールバック

manga_p2epub.exe は別リポジトリ由来のため、初回に見つからなければ
ファイル選択ダイアログで場所を尋ねて設定に保存する（リリースzipに
同梱してしまえば1のルールで自動解決される）。

## 4. 画面設計

### 4.1 メイン画面（1画面完結・ウィザードにしない）

```
┌─────────────────────────────────────────────┐
│  自炊PDF → ePub 変換                         [文字サイズ + − ] │
│                                                              │
│  ┌────────────────────────────────────────┐  │
│  │      ここにPDFをドラッグ＆ドロップ                    │  │
│  │      （または クリックしてファイルを選ぶ ）            │  │
│  └────────────────────────────────────────┘  │
│  ファイル: 霧のむこうのふしぎな町_柏葉幸子.pdf                  │
│  タイトル: [霧のむこうのふしぎな町]  著者: [柏葉幸子]           │
│                                                              │
│  本の種類:  (●) 小説（縦書き）                                │
│             ( ) 横書きの本（実用書など）                       │
│             ( ) 漫画（画像そのまま・固定レイアウト）            │
│                                                              │
│  [ ] 先に再OCRで文字を読み直す（Vision API・高精度化）          │
│                                                              │
│  ▸ 詳細設定（ふだんは触らなくてよい）                          │
│                                                              │
│  ┌──────────────┐                                    │
│  │   ▶ 変換開始      │   （実行中は ⏹ 中止 に変わる）        │
│  └──────────────┘                                    │
│  ────────────────────────────────────────    │
│  進捗: ██████████░░░░░░ 132/224ページ（再OCR中）              │
│  ログ: 本文フォントサイズ: 12.5pt 対象 222/224 ページ …        │
│                                                              │
│  完了後: [ 📖 ePubを開く ] [ 📁 フォルダを開く ]               │
│          [ ✏ テキストを校正する ] [ ↻ 校正済みからePub再生成 ] │
└─────────────────────────────────────────────┘
```

設計原則:

- **1画面・上から下へ**。ファイル→種類→実行の3操作で完了する
- タイトル・著者はファイル名 `タイトル_著者名.pdf` から自動入力（3ツール共通の
  既存規約をそのまま利用）。編集すれば `--title/--author` で上書き
- 「本の種類」で起動ツールとオプションが決まる:
  - 小説（縦書き）→ `jisui2epub --epub`
  - 横書きの本 → `jisui2epub --horizontal --ruby drop --epub`
    （--ruby drop は詳細設定で解除可能。実用書の誤検出ルビ対策の推奨既定）
  - 漫画 → `manga_p2epub`
- 再OCRチェックONなら `vision_reocr` を先に実行し、その出力
  `<入力>_vision.pdf` を続けて jisui2epub に渡す2段ジョブにする
- 前回の選択（種類・詳細設定・ウィンドウサイズ・文字サイズ）を記憶する

### 4.2 詳細設定（折りたたみ。種類に応じて表示を切替）

- 共通: 出力先フォルダ（既定=PDFと同じ場所）
- 小説/横書き: ページ範囲（`--pages`）、表紙ページ（`--cover-page`）、
  ルビ（変換する/しない=`--ruby drop`）、目次精錬OFF、画像ページ化OFF
- 漫画: 綴じ方向（右/左=`--direction`）、JPEG品質（`--quality`）、
  自動回転/自動クロップ/テキスト層のON/OFF
- 再OCR: エンジン（Vision=無料枠あり / Document AI=有料・高精度）、
  ページ範囲（`--start/--end`）、二段OCR OFF（`--no-ruby-rescue`）

### 4.3 再OCR（GCP）の設定体験

- 初回に再OCRをONにしたら、サービスアカウントJSONのファイル選択ダイアログを
  出して settings.json にパスを保存。以後は子プロセスの環境変数
  `GOOGLE_APPLICATION_CREDENTIALS` に自動セット（ユーザーが環境変数を
  触る必要をなくす）
- README_forWindows.md のトラブルシューティング項目を**エラーヒント辞書**として
  内蔵: 子プロセスの stderr に `PERMISSION_DENIED`・`billing`・
  `DefaultCredentialsError` 等を見つけたら、ログ欄の下に日本語の対処法を
  1行で表示する（例:「課金が未有効です → GCPコンソールで課金を有効化」）
- APIユニット数の報告行（`API_UNITS`）を拾い、完了時に「今月の無料枠
  1000ユニットのうち◯◯使用」を大きく表示

## 5. 実行エンジン

- `subprocess.Popen(..., stdout=PIPE, stderr=STDOUT, creationflags=CREATE_NO_WINDOW)`
  で起動。読み取りはワーカースレッド、UI反映は `queue`＋`root.after(100ms)`
  ポーリング（Tkinterはメインスレッド以外から触らない）
- 出力デコードは utf-8 → cp932 の順でフォールバック（exeのコンソール出力は
  環境でどちらもあり得る）
- **進捗バー**: ツールの既存出力を正規表現で拾う
  - vision/docai: 処理中ページ表示・チェックポイント行 → 決定的プログレス
  - jisui2epub / manga_p2epub: 数十秒で終わるためインジケータ
    （不確定バー）＋ステージ名表示で十分
- **中止ボタン**: `terminate()`。vision_reocr はチェックポイント保存
  （20ページごと）があるため中断しても安全で、次回は「続きから再開
  （--start N）」を提案する。N は GUI がログから最後のチェックポイント
  ページを拾って settings.json のジョブ履歴に記録しておく
- **2段ジョブ**（再OCR→変換）は単純な直列実行。1段目が非0終了なら停止
- 多重起動防止: 実行中は開始ボタンを無効化（キューイングはPhase 2）

## 6. 校正ループの支援（jisui2epub の従来運用をボタン化）

1. 変換完了後「✏ テキストを校正する」→ 出力 .txt を `os.startfile()` で
   既定エディタ（メモ帳等）で開く
2. 校正を終えたら「↻ 校正済みからePub再生成」→
   `jisui2epub <元PDF> --from-text <校正済みtxt>` を実行
3. 直近の変換の PDF/txt の対応は settings.json のジョブ履歴に保存してあるため、
   GUIを再起動しても後日再生成できる（履歴からの再実行リスト＝Phase 2）

## 7. 配布・ビルド

- `jisui_gui.py` を本リポジトリに追加（依存: 標準ライブラリのみ。
  ドラッグ&ドロップは `tkinterdnd2` を使い、未導入環境では自動で
  「クリックして選択」のみに退化する設計にして必須依存にしない）
- Windows側で `pyinstaller --onefile --noconsole jisui_gui.py`
  → `forwindows/jisui_gui.exe`（他exeと同じ場所に置くだけで動く）
- リリースzipに jisui_gui.exe を追加。README_forWindows.md に
  「GUIで使う場合」節を追加し、**GUI経路を推奨の入口**にする
- manga_p2epub.exe もzipに同梱するか検討（リポジトリは別のままバイナリのみ
  同梱が簡単。ライセンスは両方MITで問題なし）

## 8. 併用する最小手段（GUIを作る前でも導入可能）

`forwindows/` にドラッグ&ドロップ用バッチを3本置く:

```bat
rem 小説ePub化.bat — PDFをこのアイコンにドロップ
@echo off
"%~dp0jisui2epub.exe" %1 --epub
pause
```

（横書き用・漫画用も同様）。オプション固定・進捗なしだが実装コストゼロで、
GUI完成後も「とにかく1操作で」の代替手段として残す。

## 9. 実装フェーズ

- **Phase 1（MVP）**: メイン画面（ファイル選択・3種類・タイトル/著者自動入力・
  開始/中止・ログ欄・完了後のePub/フォルダを開く）＋設定永続化＋exe化。
  再OCRチェックと認証JSON設定も含める（進捗は不確定バーでよい）
- **Phase 2**: 決定的プログレス（ページ数パース）、再OCRの中断→再開提案、
  校正ループボタン、エラーヒント辞書、ジョブ履歴と再実行、D&D対応
- **Phase 3（必要になったら）**: 複数冊のキュー実行、本の種類の自動判別
  （縦横は既存の向き票 `count_orientation_chars`、漫画は「1ページ1画像＋
  本文級テキストなし」で判定できる見込み）、docai/vision のA/B比較支援

## 10. 実装状況（2026-07-20 Phase 1 実装済み）

`jisui_gui.py` として実装した（標準ライブラリのみ）。Phase 1 の
全項目に加え、コストが低かった Phase 2 の一部（校正ループボタン・D&D対応・
再OCRの開始/終了ページ指定）も前倒しで含めた。

- **D&Dの実装は当初案（tkinterdnd2 任意依存）から変更**: Windowsでは
  ctypes＋Win32 API（`DragAcceptFiles`＋`WM_DROPFILES` のウィンドウ
  プロシージャのサブクラス化）で**依存ゼロ**実装とした。tkinterdnd2 は
  ライブラリ未導入だと黙って無効化される上、rootを `TkinterDnD.Tk()` で
  作る必要がある（初版実装のバグ）ため、exe配布での確実性に欠けた。
  tkinterdnd2 経路は非Windows（WSLg開発環境）用に残置。管理者実行時の
  UIPI遮断は `ChangeWindowMessageFilterEx` で許可済み。ドロップ先は
  ウィンドウ全体（弱視対応: 小さな枠を狙わなくてよい）

- 検証（WSLg上）: `--smoke`（UI構築テスト）、`--print-jobs`（3種類＋再OCR
  のコマンド組み立て確認）、JobRunner の実変換テスト（合成横書きPDF→
  txt/epub 出力パスのログ回収まで確認）、`build_regen_job`・設定の
  保存/読込ラウンドトリップ
- **Windows実機で確認済み（2026-07-20）**: exe化（pyinstaller --onefile
  --noconsole）、GUI経由の横書き実本変換の完走、D&D（ドロップ→タイトル・
  著者自動入力）。実機テストで3件のバグを発見して修正した:
  1. **cp932パイプクラッシュ**: GUI（パイプ）経由で起動されたツールの
     stdoutがcp932エンコードになり、完了メッセージの✅で
     UnicodeEncodeError。GUI側=子プロセスに `PYTHONIOENCODING=utf-8:replace`
     を付与、ツール側=jisui2epub.py 冒頭で stdout/stderr を
     `errors="replace"` に再設定（vision/docaiはimport経由で同時に効く）
  2. **Win32はargtypes宣言必須**: DragFinish だけ未宣言だったため、
     HDROP（64bitポインタ）が既定の32bit int変換でOverflowErrorになり
     ドロップが無反応化。WSLではWin32経路が動かず実機でしか発現しない
  3. **ウィンドウプロシージャ内でTk API禁止**: コールバックから
     `root.after()` を呼ぶとTclイベントディスパッチへの再入となり、
     GIL状態管理と衝突して Fatal Python error (PyEval_RestoreThread) で
     即死。ドロップ結果は素のPythonリストに積み、既存の poll()
     （after 100msループ）が拾う方式に変更
- ツール解決は Windows=exe優先／開発環境=.py優先（各リポジトリの .venv を
  自動使用）。manga_p2epub は隣接リポジトリ ../mangaP2ePub も探索する。
  v1.0.0 から manga_p2epub.exe もリリースzipに同梱
- 未実装（次フェーズ）: 決定的プログレスバー（ページ数パース）、再OCR中断
  からの --start 自動提案、エラーヒント辞書、ジョブ履歴一覧、複数冊キュー、
  種類の自動判別、§8 のバッチファイル

## 11. 非目標

- ツール本体のロジック変更（GUIは公開CLIオプションの組み立てのみ）
- macOS/Linux向けGUI（開発はWSLで .py 直接起動できれば十分）
- ePubビューア機能（既存の epub_viewer / 市販リーダーの領分）
